Putting Things in Their Proper Place with CSS Position

The CSS position property is one of the most valuable assets developers have when creating web pages. With CSS positioning, we can precisely control how different block level elements display on the page. This opens up lots of creative and useful possibilities. Let’s take a few minutes to examine what CSS positioning does and how to use it.

What Does the Position Property Do?

  • It defines how an element’s location on the page is rendered in a web browser
  • It allows us to override the normal flow behaviour of an HTML document

The position property takes five possible values:

  • static
  • relative
  • absolute
  • fixed
  • sticky

The default value for the position property is static, meaning the element will behave as would be expected in the flow of the document. In this article, let’s focus on the relative, absolute, and fixed position values.

How does Position Property Work?

  • First, we must declare which element we want to position and what value we want to use.
                  .main-content {
                    position: value;
                    }
  • We then use the left, right, top, bottom, and z-index properties to determine how block level elements are displayed on the page.
                  .main-content {
                    position: value;
                    top: 20px;
                    left: 40px;
                    }
  • We can use any units to do this: pixels, mms, rems, %, etc.

Why Should We Use the CSS Position Property?

The position property allows us to do some interesting things with our layouts. Some examples of when this might be useful include the following:

  • Positioning an element to float over the top of other elements
  • To be more creative with our layout and break out of the boxes by offsetting elements
  • To have an element remain in place, such as a navigation bar

Now let’s take a closer look at the three most widely used values for CSS position.

Relative

  • The relative value changes an element’s position in the document relative to itself.
  • The units of the left, right, top, and bottom properties effectively push against the specified side of the box. This has the effect of moving it in away from what is specified. 
                          postion: relative;
                            top: 50px;
                            left: 40px;

    This will move the element away from the top by 50px and away from the left by 40px, effectively moving it down and to the right.

  • Position relative is useful when we want to offset an element from its normal flow on the page.
  • It is also useful when we want to position a child of this element absolutely. See below.

Absolute

  • When an element has position: absolute, it is displayed positioned relative to its parent elements.
  • Other elements will behave as it that element is no longer in the document i.e. they will collapse into the space the absolutely positioned element previously occupied
  • Left, top, right, and bottom determine the element’s final position. To put it another way, these properties specify the distance the element should be from its containing element’s sides.
  • Position is relative to it’s closest positioned ancestor, so the parent element MUST have a value other than static
  • If there are no positioned ancestor elements, it is positioned relative to the <html> element. In other words, it will be positioned absolutely relative to the viewport
  • Because of the above, position: absolute is almost always used in conjunction with position: relative used on its parent element.
  • Absolute position is very useful in the following situations:
    • Popup boxes
    • Control menus
    • Drag and drop UI features

Fixed

  • Like absolute positioning, position: fixed also removes an element from the normal flow of a document.
  • BUT, a fixed position element is always positioned relative to the browser viewport
  • Fixed elements are not affected by scrolling. This makes them especially useful for creating navigation elements.

So, now that you’ve had this short introduction to CSS positioning, go out and try it yourself. I think you’ll find a number of useful ways to put this property to use in your designs. Also, go have a look around the internet and see what you can do with position: sticky. Go forth and code.
