So you want to learn about web development but you don’t know the first thing about it? Hopefully, by the end of this post, you’ll have gained some familiarity with some of the languages and techniques used by web developers today.

Though technically not programming languages, HTML (Hypertext Markup Language) and CSS (Cascading Style Sheets) are used to create a user sees when they look at a web page. HTML is like a house. It provides the foundation, the walls, and the roof. It provides the basic furniture needed to make the house usable and can also hang some pictures on the wall. But without CSS, it is a just a drab cinderblock building devoid of much colour or personality. Like this.

See the Pen Simple HTML Example 2 by John Gray (@meetjohngray) on CodePen.

That’s where CSS comes in. It is the professional decorator that comes in and paints or wall papers the walls. It decides on how to arrange the furnishings and perhaps swaps them out for a different style or adds new ones. CSS might even decide to rearrange the rooms of the house, creating a better layout and flow. CSS can take the same house built by HTML and transform its look in innumerable ways. But what if you want your web page to be more like a smart home? One that opens the doors as you get near them, or responds in other ways to your requests. That’s where JavaScript (JS) comes in. JS allows developers to add interactivity to web pages. It does this through using a variety of techniques including utilising control flow and loops.

Control flow allows JavaScript to do a series of things while making logical decisions about how to proceed. One way it does this is through If/Else statements. This is a technique all of do everyday without even thinking about it. If I walk into my flat and the temperature is too cold, I will turn on the heater and put on a warm hat. If the window is open, I will close it. But if the room is too hot, I will turn on the fan. Else if the temperature is just right, I won’t do anything at all.

Loops help us (and JS) repeat a series of steps until reaching a specified result. When I lose my keys in the house, I loop through all the places I normally misplace them. On top of the refrigerator, in the pocket of the pair of pants I wore yesterday, etc. I do this until I find my keys.

JavaScript uses arrays and objects to store the information it needs to perform the tasks we ask of it. Arrays can be used to store simple collections of data, such as items entered into a grocery shopping list.

let shopping_list = [bananas, apples, oatmeal, tofu, hummus, chick peas]

Starting from zero, each item in the list is automatically assigned a sequential number by JS. These numbers make up an index. The index can be used to access each item (value). Using the above example, bananas would have an index of 0, apples would be 1, etc.

Though useful, this method of accessing information is not the most user friendly or versatile. That’s where objects come in. Objects allow developers to create collections of data utilising logical indexes. This allows them to call the value they are looking for by name. An object could be used to contain information for a more complex shopping list. This list could be indexed according to what aisle of the store the items are located. Below is an example of what this object could look like.

let shopping_list = {
                  'produce': 'bananas', 'apples',
                  'dried goods': 'oatmeal', 'chick peas',
                  'refrigerated': 'tofu', 'hummus' 
                  }

Now that we have the information we need stored, functions can be used to help us access it. Functions are pieces of code that perform repeated tasks, preventing developers from having to rewrite the same code over and over. Below is a function that allows us to access the information from the object we created above.

function get_items(list, category) {
              console.log(list[category])
            }
            
            get_items(shopping_list,'produce')

Result: 

["bananas", "apples"]

This function takes the shopping_list object and prints out a list of just the items specified by the category argument.

HTML, CSS, and JavaScript do what they do by interacting with the Document Object Model (DOM). The DOM is the builder. It takes the blueprint provided by HTML and uses a web browser to construct a webpage. In so doing it creates a tree-like structure where each branch ends in a node. These nodes are the elements that make up the house. Some are visible and some are not, like the wiring, plumbing, and duct work. CSS and JS use the DOM as the framework on which to hang their styles and interactions respectively.
